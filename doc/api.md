## 1. 概述

### 1.1 基本信息
- **基础URL**: `https://api.yourdomain.com/api`（示例，请替换为实际域名）
- **协议**: HTTPS
- **字符编码**: UTF-8
- **内容类型**: 请求与响应均使用 `application/json`（特殊说明除外）
- **版本**: v1.0.0-beta

### 1.2 公共响应结构
所有接口返回的JSON格式统一如下：

| 字段 | 类型 | 必含 | 描述 |
|------|------|------|------|
| `code` | int | 是 | 业务状态码，200表示成功，其他表示失败 |
| `msg` | string | 是 | 响应消息，可用于调试，不应直接展示给用户 |
| `data` | object/array/string | 否 | 响应数据，具体结构见各接口说明 |

**示例**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

### 1.3 通用状态码
| 状态码 | 说明 | 描述 |
|--------|------|------|
| 200 | Success | 请求成功 |
| 400 | Bad Request | 请求参数错误（如缺少必填字段、格式错误） |
| 401 | Unauthorized | 未认证或会话已过期 |
| 403 | Forbidden | 权限不足，禁止访问 |
| 404 | Not Found | 请求的资源不存在 |
| 500 | Internal Server Error | 服务器内部错误，请稍后重试 |

> **注意**：各接口可能返回更具体的业务错误码，详见接口说明中的“错误码”部分。

### 1.4 认证方式
- **Session/Cookie 认证**：用户登录成功后，服务器会下发 `JSESSIONID` 写入 Cookie，后续请求需携带该 Cookie。
- **无需认证的接口**：登录接口（`/user/login`）无需认证。
- **需要认证的接口**：除登录外，所有接口均需携带有效的会话凭证。

### 1.5 数据格式说明
- **时间格式**：所有时间字段均采用 **ISO 8601** 格式，例如 `2026-03-12T10:30:00.000Z`（UTC 时间）。
- **ID字段**：均为整数类型（`Integer`）。
- **删除标识**：`isDeleted` 字段，0-未删除，1-已删除。

## 2. 用户相关接口

### 2.1 获取用户信息

#### 接口说明
获取指定用户的基本信息。若不传 `userId`，默认返回当前登录用户信息（需认证）。

- **请求URL**: `/user/info`
- **请求方法**: `GET`
- **Content-Type**: `application/x-www-form-urlencoded`（参数拼接在URL上）
- **认证要求**: 需要登录

#### 请求参数
| 参数名 | 类型 | 必填 | 示例 | 描述 |
|--------|------|------|------|------|
| `userId` | int | 否 | 1 | 用户ID，默认为当前登录用户的ID |

#### 响应参数
| 参数名 | 类型 | 必含 | 示例 | 描述 |
|--------|------|------|------|------|
| `code` | int | 是 | 200 | 业务状态码 |
| `msg` | string | 是 | "操作成功" | 响应消息 |
| `data` | object | 是 | 见下方 | 用户信息对象 |

**data 对象结构**：
| 字段 | 类型 | 必含 | 示例 | 描述 |
|------|------|------|------|------|
| `id` | int | 是 | 1 | 用户ID |
| `username` | string | 是 | "admin" | 用户名 |
| `password` | string | 是 | "$2a$10$..." | BCrypt加密后的密码（通常不返回，但当前版本返回） |
| `avatar` | string | 否 | null | 头像URL |
| `createTime` | string | 是 | "2024-03-06T08:11:00.000Z" | 创建时间 |
| `updateTime` | string | 是 | "2024-03-06T08:11:00.000Z" | 更新时间 |
| `isDeleted` | int | 是 | 0 | 删除标识（0-未删除，1-已删除） |

#### 请求示例
```http
GET /api/user/info?userId=1 HTTP/1.1
Host: api.yourdomain.com
Cookie: JSESSIONID=xxx
```

#### 响应示例
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "id": 1,
    "username": "admin",
    "password": "$2a$10$NQVgZc5sQB7FvHMRxJrwkedBqMTMtwL0C2YdytKE.Ur9eyo9ydwYm",
    "avatar": null,
    "createTime": "2024-03-06T08:11:00.000Z",
    "updateTime": "2024-03-06T08:11:00.000Z",
    "isDeleted": 0
  }
}
```

#### 错误码
| 错误码 | 说明 | 描述 |
|--------|------|------|
| 200 | 成功 | 获取用户信息成功 |
| 404 | 用户不存在 | 指定的 `userId` 不存在 |
| 500 | 服务器内部错误 | 获取用户信息失败 |

### 2.2 用户登录

#### 接口说明
用户登录认证，成功后服务器会设置会话Cookie。

- **请求URL**: `/user/login`
- **请求方法**: `POST`
- **Content-Type**: `application/json`
- **认证要求**: 无需认证

#### 请求参数
| 参数名 | 类型 | 必填 | 示例 | 描述 |
|--------|------|------|------|------|
| `username` | string | 是 | "admin" | 用户名 |
| `password` | string | 是 | "password" | 密码 |

#### 请求体示例
```json
{
  "username": "admin",
  "password": "password"
}
```

#### 响应参数
| 参数名 | 类型 | 必含 | 示例 | 描述 |
|--------|------|------|------|------|
| `code` | int | 是 | 200 | 业务状态码 |
| `msg` | string | 是 | "操作成功" | 响应消息 |
| `data` | object | 是 | 见下方 | 登录结果数据 |

**data 对象结构**：
| 字段 | 类型 | 必含 | 描述 |
|------|------|------|------|
| `user` | object | 是 | 用户信息对象（同 2.1 节 data 结构） |

#### 响应示例
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "user": {
      "id": 1,
      "username": "admin",
      "password": "$2a$10$NQVgZc5sQB7FvHMRxJrwkedBqMTMtwL0C2YdytKE.Ur9eyo9ydwYm",
      "avatar": null,
      "createTime": "2024-03-06T08:11:00.000Z",
      "updateTime": "2024-03-06T08:11:00.000Z",
      "isDeleted": 0
    }
  }
}
```

#### 错误码
| 错误码 | 说明 | 描述 |
|--------|------|------|
| 200 | 成功 | 登录成功 |
| 400 | 请求参数错误 | 用户名或密码为空 |
| 401 | 认证失败 | 用户名不存在或密码错误 |
| 500 | 服务器内部错误 | 登录失败 |

## 3. 打卡相关接口

### 3.1 获取打卡记录（分页）

#### 接口说明
获取指定用户的打卡记录，支持分页查询。

- **请求URL**: `/punch/record`
- **请求方法**: `GET`
- **Content-Type**: `application/x-www-form-urlencoded`（参数拼接在URL上）
- **认证要求**: 需要登录

#### 请求参数
| 参数名 | 类型 | 必填 | 示例 | 默认值 | 描述 |
|--------|------|------|------|--------|------|
| `userId` | int | 是 | 1 | - | 用户ID |
| `page` | int | 否 | 1 | 1 | 当前页码，从1开始 |
| `size` | int | 否 | 15 | 15 | 每页记录数，最大不超过100 |

#### 响应参数
| 参数名 | 类型 | 必含 | 示例 | 描述 |
|--------|------|------|------|------|
| `code` | int | 是 | 200 | 业务状态码 |
| `msg` | string | 是 | "操作成功" | 响应消息 |
| `data` | object | 是 | 见下方 | 分页数据对象 |

**data 对象结构**：
| 字段 | 类型 | 必含 | 描述 |
|------|------|------|------|
| `records` | array | 是 | 打卡记录列表，每个元素结构见下表 |
| `total` | int | 是 | 总记录数 |
| `page` | int | 是 | 当前页码 |
| `size` | int | 是 | 每页条数 |
| `pages` | int | 是 | 总页数 |

**records 数组元素结构**：
| 字段 | 类型 | 必含 | 示例 | 描述 |
|------|------|------|------|------|
| `id` | int | 是 | 1 | 记录ID |
| `userId` | int | 是 | 1 | 用户ID |
| `checkInTime` | string | 是 | "2024-03-06T08:11:00.000Z" | 打卡时间（UTC） |
| `checkInType` | int | 是 | 1 | 打卡类型：1-上班打卡，2-下班打卡（预留） |
| `checkInStatus` | int | 是 | 1 | 打卡状态：1-正常，2-迟到，3-早退（预留） |
| `checkInLocation` | string | 是 | "公司" | 打卡地点 |
| `createTime` | string | 是 | "2024-03-06T08:11:00.000Z" | 创建时间 |
| `updateTime` | string | 是 | "2024-03-06T08:11:00.000Z" | 更新时间 |
| `isDeleted` | int | 是 | 0 | 删除标识 |

#### 请求示例
```http
GET /api/punch/record?userId=1&page=1&size=15 HTTP/1.1
Host: api.yourdomain.com
Cookie: JSESSIONID=xxx
```

#### 响应示例
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "records": [
      {
        "id": 1,
        "userId": 1,
        "checkInTime": "2024-03-06T08:11:00.000Z",
        "checkInType": 1,
        "checkInStatus": 1,
        "checkInLocation": "公司",
        "createTime": "2024-03-06T08:11:00.000Z",
        "updateTime": "2024-03-06T08:11:00.000Z",
        "isDeleted": 0
      }
    ],
    "total": 1,
    "page": 1,
    "size": 15,
    "pages": 1
  }
}
```

#### 错误码
| 错误码 | 说明 | 描述 |
|--------|------|------|
| 200 | 成功 | 获取打卡记录成功 |
| 500 | 服务器内部错误 | 获取打卡记录失败 |

用户进行上班打卡操作。后端会根据打卡时间自动判断状态（正常/迟到）。

- **Content-Type**: `application/json`
- **认证要求**: 需要登录

#### 请求参数
| 参数名 | 类型 | 必填 | 示例 | 描述 |
|--------|------|------|------|------|
| `username` | string | 是 | "admin" | 用户名 |
| `punchTime` | string | 是 | "2024-03-06T08:11:00.000Z" | 打卡时间（UTC） |
| `userId` | int | 是 | 1 | 用户ID |

#### 请求体示例
```json
{
  "username": "admin",
  "punchTime": "2024-03-06T08:11:00.000Z",
  "userId": 1
}
```

#### 响应参数
| 参数名 | 类型 | 必含 | 示例 | 描述 |
|--------|------|------|------|------|
| `code` | int | 是 | 200 | 业务状态码 |
| `msg` | string | 是 | "打卡成功" | 响应消息 |
| `data` | null | 否 | null | 无数据 |

#### 请求示例
```http
POST /api/punch/in HTTP/1.1
Host: api.yourdomain.com
Cookie: JSESSIONID=xxx
Content-Type: application/json

{
  "username": "admin",
  "punchTime": "2024-03-06T08:11:00.000Z",
  "userId": 1
}
```

#### 响应示例
```json
{
  "code": 200,
  "msg": "打卡成功",
  "data": null
}
```

#### 错误码
| 错误码 | 说明 | 描述 |
|--------|------|------|
| 200 | 成功 | 打卡成功 |
| 400 | 请求参数错误 | 用户ID或打卡时间为空 |
| 404 | 用户不存在 | 指定的 `userId` 不存在 |
| 500 | 服务器内部错误 | 打卡失败 |

## 4. 附录

### 4.1 时间格式说明
所有时间字段均使用 **UTC 时间**，格式为 `yyyy-MM-dd'T'HH:mm:ss.SSS'Z'`（ISO 8601）。例如：
- `2026-03-12T10:30:00.000Z` 表示 2026年3月12日 10:30:00 UTC。

### 4.2 错误处理建议
- 客户端应首先检查 `code` 字段，`code === 200` 表示成功，否则表示失败。
- `msg` 字段内容可能变化，不应依赖其具体文本进行业务判断。
- 对于分页接口，当 `page` 超出总页数时，`records` 返回空数组，`code` 仍为 200。
- 网络或服务器异常时，可能无法收到 JSON 响应，客户端应做好超时和重试机制。
