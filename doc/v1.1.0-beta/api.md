# v1.1.0-beta API 接口文档

**版本**: v1.1.0-beta  
**更新时间**: 2026-03-13  
**状态**: 设计中  

## 1. 接口概述

### 1.1 基本信息
- **协议**: HTTPS
- **域名**: `https://api.yourdomain.com`
- **版本**: v1.1.0-beta
- **字符编码**: UTF-8
- **内容格式**: JSON

### 1.2 公共请求参数
无特殊公共请求参数

### 1.3 公共响应参数
| 字段名 | 类型   | 是否必须 | 描述       |
|--------|--------|----------|------------|
| code   | Int    | 是       | 业务状态码 |
| msg    | String | 是       | 响应消息   |
| data   | Object | 否       | 响应数据   |

### 1.4 通用状态码
| 状态码 | 说明          | 描述           |
|--------|---------------|----------------|
| 200    | Success       | 操作成功       |
| 400    | Bad Request   | 请求参数错误   |
| 401    | Unauthorized  | 未授权访问     |
| 403    | Forbidden     | 权限不足       |
| 404    | Not Found     | 资源不存在     |
| 500    | Server Error  | 服务器内部错误 |

## 2. 部门管理相关接口

### 2.1 创建部门

#### 接口描述
创建新的部门信息

#### 请求信息
- **请求地址**: `/api/department/create`
- **请求方法**: POST
- **内容类型**: application/json
- **认证方式**: JWT Token

#### 请求参数
| 参数名 | 类型   | 是否必须 | 示例值     | 描述         |
|--------|--------|----------|------------|--------------|
| name   | String | 是       | "技术部"   | 部门名称     |
| description | String | 否   | "技术研发部门" | 部门描述   |
| parentId | Integer | 否     | 0          | 父部门ID，顶级部门为0 |

#### 请求体示例
```json
{
  "name": "技术部",
  "description": "技术研发部门",
  "parentId": 0
}
```

#### 响应参数
| 参数名 | 类型    | 是否必须 | 示例值      | 描述         |
|--------|---------|----------|-------------|--------------|
| code   | Integer | 是       | 200         | 业务状态码   |
| msg    | String  | 是       | "创建成功"  | 响应消息     |
| data   | Object  | 否       | -           | 部门信息对象 |

#### 响应示例
```json
{
  "code": 200,
  "msg": "创建成功",
  "data": {
    "id": 1,
    "name": "技术部",
    "description": "技术研发部门",
    "parentId": 0,
    "createTime": "2024-03-13T10:00:00.000+00:00",
    "updateTime": "2024-03-13T10:00:00.000+00:00"
  }
}
```

#### 错误码
| 错误码 | 说明         | 描述             |
|--------|--------------|------------------|
| 200    | Success      | 创建部门成功     |
| 400    | Bad Request  | 参数验证失败     |
| 401    | Unauthorized | 未授权访问       |
| 403    | Forbidden    | 权限不足         |
| 500    | Server Error | 创建部门失败     |

### 2.2 获取部门列表

#### 接口描述
获取所有部门的列表（支持分页）

#### 请求信息
- **请求地址**: `/api/department/list`
- **请求方法**: GET
- **内容类型**: application/x-www-form-urlencoded
- **认证方式**: JWT Token

#### 请求参数
| 参数名 | 类型    | 是否必须 | 示例值 | 默认值 | 描述     |
|--------|---------|----------|--------|--------|----------|
| page   | Integer | 否       | 1      | 1      | 页码     |
| size   | Integer | 否       | 15     | 15     | 每页条数 |

#### 响应参数
| 参数名 | 类型    | 是否必须 | 示例值      | 描述         |
|--------|---------|----------|-------------|--------------|
| code   | Integer | 是       | 200         | 业务状态码   |
| msg    | String  | 是       | "操作成功"  | 响应消息     |
| data   | Object  | 否       | -           | 分页数据对象 |

##### data 详细参数
| 参数名 | 类型    | 是否必须 | 示例值 | 描述         |
|--------|---------|----------|--------|--------------|
| departments | Array   | 是       | []     | 部门列表     |
| total  | Integer | 是       | 1      | 总记录数     |
| page   | Integer | 是       | 1      | 当前页码     |
| size   | Integer | 是       | 15     | 每页条数     |
| pages  | Integer | 是       | 1      | 总页数       |

#### 响应示例
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "departments": [
      {
        "id": 1,
        "name": "技术部",
        "description": "技术研发部门",
        "parentId": 0,
        "createTime": "2024-03-13T10:00:00.000+00:00",
        "updateTime": "2024-03-13T10:00:00.000+00:00"
      }
    ],
    "total": 1,
    "page": 1,
    "size": 15,
    "pages": 1
  }
}
```

## 5. 权限验证接口

### 5.1 验证用户权限

#### 接口描述
验证当前用户是否具有指定权限

#### 请求信息
- **请求地址**: `/api/auth/check-permission`
- **请求方法**: POST
- **内容类型**: application/json
- **认证方式**: JWT Token

#### 请求参数
| 参数名 | 类型   | 是否必须 | 示例值     | 描述         |
|--------|--------|----------|------------|--------------|
| permission | String | 是   | "user:read"| 权限标识     |

#### 请求体示例
```json
{
  "permission": "user:read"
}
```

#### 响应参数
| 参数名 | 类型    | 是否必须 | 示例值      | 描述         |
|--------|---------|----------|-------------|--------------|
| code   | Integer | 是       | 200         | 业务状态码   |
| msg    | String  | 是       | "验证通过"  | 响应消息     |
| data   | Object  | 否       | -           | 验证结果对象 |

#### 响应示例
```json
{
  "code": 200,
  "msg": "验证通过",
  "data": {
    "hasPermission": true,
    "userId": 1
  }
}
```
