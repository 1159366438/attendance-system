# 变更日志

## [v1.1.0] - 2026-03-15

### ✨ 新增功能
- **用户注册功能**
  - 前端：实现表单验证（用户名、密码、年龄、性别、头像选择）
  - 后端：完成数据存储与验证，确保用户名唯一性
  - 性别选项支持（男、女、未知）
- **自动密码哈希修复机制**
  - 系统启动时自动检测并修复数据库中不正确的 BCrypt 密码哈希值，确保旧版本数据兼容性和默认账号（admin/123456）可用性
  - 通过 `PasswordFixer.java` 实现，无需手动干预，提升系统健壮性与用户体验

### 🐛 Bug 修复
#### 前端
- 修复登录/注册页面布局问题，不再显示侧边栏
- 修复 TypeScript 类型错误，补充缺失的常量定义
- 清理未使用的导入，消除 TypeScript 警告
- 修复登录页面在已登录状态下仍发送不必要的用户信息请求的问题
- 修复硬编码路径字符串问题，统一使用路由常量

#### 后端
- 解决端口冲突，将后端服务端口从 8080 调整为 8081
- 统一登录和注册的验证逻辑，消除代码重复
- 移除 Service 层中重复的验证逻辑
- 统一错误处理风格为 `ResponseResult` 模式
- 修复性别字段未正确传递到后端的问题

### 🔧 重构与代码质量
- **常量管理**：
  - 将表单验证相关类型提取为常量
  - 使用 `STORAGE_VALUES` 管理所有存储键名
  - 将用户名/密码长度限制等硬编码数值替换为常量
  - 将硬编码布尔值替换为 `BOOLEAN_CONSTANTS`
  - 将路径字符串替换为 `ROUTE_CONSTANTS`
- **样式优化**：
  - 将注册页面样式提取到独立的 CSS 文件中，与主样式解耦
- **数据库文档与SQL**：
  - 将数据库文档按功能模块拆分为多个文件，便于维护
  - 重构数据库初始化脚本，按模块化方式组织SQL DDL语句
  - 整合所有数据库表结构到统一的schema.sql文件中，包含用户、打卡、部门、角色、权限等模块
- **代码规范**：
  - 为后端 Java 文件添加标准文件头注释，包括 @author 和 @since 标签
